# 北洋闪传

`北洋闪传` 是根据 `lec22 网络编程.pdf` 中“挑战2：文件速传”要求完成的文件速传项目。Android APK 使用 Kivy 原生界面；电脑端运行 `main.py` 会启动一个美观的网页控制台，操作步骤与手机端保持一致。

## 功能完成情况

- 发送方可以选择本地文件。
- 接收方可以选择文件保存位置。
- Android 端保存位置支持预设目录和手动路径，保存弹窗已针对窄屏优化。
- 电脑端支持网页控制台，流程与手机端一致：开始接收、输入对方 IP、选择文件、立即发送。
- 支持发送前 ZIP 压缩。
- 支持接收后自动解压 ZIP。
- 手机端和电脑端都使用 TCP `50022` 进行文件直传，64 KB 分块传输。
- 发送方和接收方都显示进度、速度、剩余时间。
- 双方都可以取消当前传输。
- 支持断点续传：未完成内容保存为 `.part`，状态保存为 `.state.json`。
- 支持 SHA-256 文件签名校验，校验通过后才生成最终文件。
- 已增强 TCP keepalive、传输缓冲区、传输阶段超时和校验等待时间，减少大文件或慢网络下误判失败。
- 发送端连接失败会自动短暂重试，支持输入 `IP` 或 `IP:端口`。
- 对方 IP 会自动保存，App 或电脑网页重启后仍会保留上次输入。
- Android 发送端默认使用“文档安全选择页”：App 在手机本机启动 `127.0.0.1` 上传页，由浏览器选择 PDF、Word、ZIP 等文档并传回 App，绕开 Kivy/pyjnius 直接读取文档 Provider 时可能闪退的问题。
- Android 接收完成后会主动刷新系统文件索引，让文件管理器更快显示新文件。
- 支持手机和桌面运行，适合同一局域网内面对面同时在线直传。

## Android 保存位置

Android 系统对文件夹选择和文件索引有权限限制，Kivy 内置文件夹选择器在部分手机上会显示异常或无法进入真实目录。因此本项目在 Android 上使用专门的保存位置面板。新版默认优先保存到公共下载目录，方便在文件管理器中找到：

- 下载/北洋闪传（推荐）：`/storage/emulated/0/Download/北洋闪传`
- 文档/北洋闪传：`/storage/emulated/0/Documents/北洋闪传`
- 相册/北洋闪传：`/storage/emulated/0/DCIM/北洋闪传`
- 下载根目录：`/storage/emulated/0/Download`
- 应用专用目录（最稳但可能隐藏）：`/storage/emulated/0/Android/data/org.tju.challenge.beiyangflashtransfer/files/北洋闪传`
- 手动路径：可自行输入可写路径

点击保存位置后，程序会尝试创建目录并写入一个临时测试文件。如果该目录不可写，会在运行记录中提示错误，避免传输完成后才发现保存失败。

接收完成后，程序会执行两件事：

- 强制把接收到的文件写入磁盘，降低“显示完成但文件还没落盘”的概率。
- 通知 Android 刷新媒体/文件索引，让文件管理器更快看到文件。

如果你选择了 `Android/data/...` 这类应用专用目录，部分 Android 11+ 手机的系统文件管理器会隐藏这个目录，这是系统限制，不是传输失败。建议优先使用“下载/北洋闪传”。

## 使用流程

1. 两台设备连接同一个局域网或同一个手机热点。
2. 接收方打开 App，查看首页显示的“我的 IP”。
3. 接收方选择保存位置，然后点击“开始接收”。
4. 发送方打开 App，在“对方 IP”里输入接收方 IP。
5. 发送方点击“选择文件”，可按需勾选“发送前压缩 ZIP”。
6. 发送方点击“立即发送”。
7. 传输中双方都能看到进度、速度、剩余时间，也可以点击“取消传输”。
8. 如果网络断开或主动取消，再次传输同一个文件会从断点继续。

对方 IP 只需要输入一次。之后只要你不改输入框，App 重启后仍会保留上次的 IP；电脑网页端也会在浏览器中记住上次输入。

Android 手机端选择 PDF、Word 等文档时，点击“选择文件”会打开一个本机文档安全选择页。请在该页面选中文件并点击“使用这个文件”，页面提示成功后切回北洋闪传 App，再点击“立即发送”。

## 校园网传输说明

本程序使用 TCP 直连，默认端口为 `50022`。如果手机热点、宿舍路由器或普通局域网可以传输，但校园 Wi-Fi 不能传输，通常不是 App 代码本身的问题，而是校园网开启了以下限制：

- 客户端隔离：同一个 Wi-Fi 下的两台设备不能互相访问。
- 入站 TCP 限制：校园网允许上网，但阻止别人连接到你的手机。
- 多网卡路由混乱：手机同时开着移动数据和 Wi-Fi，发送方输入的不是当前 Wi-Fi IP。

可行处理方式：

- 两台设备连接同一个手机热点，这是最稳定的测试方式。
- 使用同一个不隔离的路由器或局域网。
- 关闭移动数据，只保留 Wi-Fi 后再刷新 IP。
- 确认发送方输入的是接收方 App 中显示的 Wi-Fi IP。
- 如果校园网强制客户端隔离，纯局域网直连 App 无法绕过，需要中继服务器或云端转发。

## 电脑网页运行

先安装依赖：

```bash
pip install -r requirements.txt
```

运行：

```bash
python main.py
```

程序会启动网页控制台，并在终端输出类似：

```text
北洋闪传电脑端已启动
网页控制台：http://127.0.0.1:50023
同一局域网设备打开：
  http://你的电脑IP:50023
文件传输 TCP 端口：50022
```

电脑端网页和手机端 APK 使用相同思路：

1. 接收方先点“开始接收”。
2. 发送方在“对方 IP”中输入接收方显示的 IP。
3. 发送方点击“选择文件”。
4. 可按需勾选“发送前压缩 ZIP”。
5. 点击“立即发送”。
6. 双方都能看到进度、速度和剩余时间。

电脑作为接收方时：电脑运行 `python main.py`，在网页里点“开始接收”；手机端输入电脑网页中显示的 IP 后发送。

电脑作为发送方时：手机端先点“开始接收”；电脑网页输入手机端显示的 IP，选择文件后点“立即发送”。

网页控制台端口默认是 `50023`，文件传输端口仍是 `50022`。这样网页界面和文件传输不会互相占用端口。

如果还想在电脑上打开旧的 Kivy 原生窗口，可以运行：

```bash
python main.py --kivy
```

## APK 构建

### 最简单方法：GitHub Actions 云端生成

这条路线不需要在你的电脑上安装 Java、Android SDK、NDK 或 Buildozer。

1. 新建一个 GitHub 仓库。
2. 把本项目所有文件上传到仓库根目录，注意保留 `.github/workflows/build-apk.yml`。
3. 打开仓库页面的 `Actions`。
4. 选择 `Build Android APK`。
5. 点击 `Run workflow`，或直接上传文件触发自动构建。
6. 等待完成后，在运行结果页面的 `Artifacts` 下载 `BeiyangFlash-debug-apk`。
7. 解压后即可得到 `.apk` 安装包。

第一次云端构建通常需要 20 到 60 分钟，因为要下载 Android SDK/NDK 和编译依赖。之后有缓存时会更快。

### 本地方法：Linux / WSL 构建

Kivy 打 APK 通常需要 Linux 或 WSL 环境，并安装 Android SDK/NDK、Java 和 Buildozer。项目已经附带 `buildozer.spec` 和 `build_apk_linux.sh`。

在 Linux / WSL 中进入本目录后执行：

```bash
chmod +x build_apk_linux.sh
./build_apk_linux.sh
```

生成的 APK 会在：

```text
bin/*.apk
```

## 协议说明

程序使用 TCP 连接，默认端口为 `50022`。

发送方先发送 4 字节大端序整数，表示后面 JSON 元数据长度。随后发送：

```json
{
  "op": "offer",
  "filename": "video.mp4",
  "size": 104857600,
  "sha256": "文件SHA-256签名",
  "compressed": false,
  "chunk_size": 65536
}
```

接收方根据 `.part` 和 `.state.json` 判断能否断点续传，然后返回：

```json
{
  "ok": true,
  "offset": 5242880
}
```

发送方从 `offset` 指定的位置继续发送。接收完成后，接收方计算 SHA-256；签名一致才把 `.part` 改名为最终文件，并向发送方返回校验通过结果。

## PDF 要求对应表

| PDF 挑战要求 | 本项目实现 |
| --- | --- |
| 发送方可以选择要发送的文件 | APK：“选择文件”按钮；电脑网页：点击/拖拽选择文件 |
| 接收方可以选择文件保存的位置 | APK：“保存位置”按钮；电脑网页：保存目录输入框 |
| 文件压缩 / 解压缩 | “发送前压缩 ZIP”“接收后自动解压” |
| 大文件传送时双方看到进度 | 进度条 + 百分比 + 速度 + 剩余时间 |
| 双方可以取消文件传输 | APK：“取消传输”按钮；网页可中止当前上传 |
| 支持断点续传 | `.part` + `.state.json` + offset 协商 |
| 支持文件校验 | SHA-256 文件签名 |
| 网络：校园网 | TCP 局域网直连；若校园网隔离客户端，需要换热点/不隔离局域网 |
| 设备：手机、桌面 | Android APK + 电脑网页模式 |
| 连接建立方式：面对面同时在线 | 接收方监听，发送方输入接收方 IP；电脑网页与手机端流程一致 |
| 小文件传送期间不支持 IP 地址改变 | 传输中固定 TCP/HTTP 连接 |
